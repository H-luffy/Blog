# 个人博客从零搭建与手写实战指南

> 本文档整理了从“使用现成工具搭建博客”到“使用 Vue3 + Spring Boot 手写博客”的完整指南，旨在帮助你打造属于自己的数字花园，并以此作为全栈开发的练手项目。

---

## 第一部分：从零开始做个人博客的通用建议

搭建个人博客是一件非常有成就感的事情，它不仅是你的“数字名片”，更是你思考的沉淀和知识的复利工具。为了避免“在折腾技术中迷失，最终放弃了写作”，请参考以下指南。

### 1. 明确博客定位
在动手之前，先问自己三个问题：
- **写给谁看？** 自己（私密笔记）、同行（技术/专业分享）、大众（生活/观点）。
- **核心内容是什么？** 技术教程、读书笔记、生活随笔、摄影作品？
- **你的技术水平如何？** 纯小白、懂一点前端/代码、资深开发者？

### 2. 选择技术方案
根据你的技术水平和需求，有三条主流路线：

#### 路线 A：零代码/低代码平台（适合纯小白、专注内容）
- **工具**：Notion + NotionNext / Super、Substack / 竹白、Halo (云端版) / WordPress.com。
- **优点**：开箱即用，排版极佳，无需维护服务器。
- **缺点**：定制化程度低，部分高级功能需付费，数据不在自己手里。

#### 路线 B：静态站点生成器 SSG（适合程序员、极客、最推荐）
- **工具**：Hugo（速度极快）、Hexo（中文生态好）、Astro / VitePress（前端现代轻量）。
- **优点**：完全免费（可白嫖托管平台）、加载极快、不用维护数据库、绝对安全。使用 Markdown 写作。
- **缺点**：需要懂一点命令行和 Git，没有可视化的后台管理界面。

#### 路线 C：动态博客系统 CMS（适合需要强大后台、有运维能力）
- **工具**：WordPress（插件无穷）、Typecho（极简轻量）、Halo (自建，界面漂亮)、Ghost（适合付费订阅）。
- **优点**：有完整的后台管理，所见即所得，功能强大。
- **缺点**：需要购买服务器和数据库，需要自己维护安全和更新，有金钱成本。

### 3. 基础设施（域名与托管）
- **购买域名**：首选 `.com`，其次 `.me`、`.io`、`.dev`。尽量短且好记。渠道：NameSilo、Cloudflare、阿里云/腾讯云。
- **选择托管**：
  - **静态博客**：代码推送到 GitHub，使用 Vercel、Netlify 或 Cloudflare Pages 部署（完全免费且自带全球 CDN）。
  - **动态博客**：购买轻量应用服务器（腾讯云/阿里云等），或使用 Zeabur、Railway 等 PaaS 平台部署。

### 4. 设计与用户体验 (UI/UX)
- **主题选择**：不要自己从零写 CSS，去 GitHub 找成熟的开源主题。
- **核心原则**：阅读体验第一（字体适中、行高 1.6-1.8、支持暗黑模式）；速度至上（减少花哨动画）。
- **必备组件**：评论系统（Giscus、Twikoo）、搜索功能、文章目录 (TOC)。

### 5. 内容创作与推广
- **写作工具**：推荐 Obsidian、Typora 或 Logseq。
- **图床避坑**：不要用免费公共图床。推荐 Cloudflare R2、阿里云/腾讯云 OSS，或直接把图片和文章放在同一个 Git 仓库。
- **SEO 与推广**：配置 `sitemap.xml`，提交给 Google Search Console。将文章同步到知乎、掘金、公众号等平台引流。加入“开往”、“博友圈”等互联项目。

### 💡 给新手的 5 个“避坑”忠告
1. **警惕“差生文具多”**：先写内容，再折腾样式！
2. **做好备份**：静态博客用 Git 推送，动态博客定期备份数据库。
3. **不要有更新压力**：个人博客不需要日更，高质量长文胜过流水账。
4. **拥抱长期主义**：坚持 2-3 年，它会成为极具价值的个人数字资产。
5. **保持真实**：写你真正关心的东西，真实的个人视角是 AI 无法替代的。

## 第二部分：自己手写博客练手的实战指南

使用现成系统是为了“快速拥有”，而**自己从零写一个博客，是为了“掌握”Web 开发的核心技能。** 这是全栈开发中最经典的练手项目。

### 1. 定义 MVP（最小可行性产品）
第一版千万不要做评论、点赞、复杂的标签分类。V1.0 只需要实现：
1. **文章列表页**：展示标题、摘要、时间，支持分页。
2. **文章详情页**：将 Markdown 渲染成 HTML。
3. **后台管理**：简单的表单，输入标题和 Markdown 内容并发布。
4. **后台登录**：简单的密码验证。

### 2. 核心数据库设计
无论用什么语言，V1.0 只需要一张核心表 `posts`：
- `id` (主键)
- `title` (文章标题)
- `summary` (文章摘要)
- `content` (Markdown 原文)
- `status` (状态: 0-草稿, 1-已发布)
- `created_at` / `updated_at` (时间戳)

### 3. 开发 Roadmap
- **阶段 1：后端与 API**。实现文章的 CRUD 接口和简单的 JWT 登录接口。
- **阶段 2：前端页面**。配置路由，调用 API，引入成熟的 Markdown 渲染库（如 `markdown-it`）和代码高亮库。
- **阶段 3：联调与优化**。处理 Loading 状态、错误提示、添加路由守卫。
- **阶段 4：部署上线**。前端部署到 Vercel，后端部署到云服务器。

### 💡 手写博客的 4 个避坑忠告
1. **图片千万别存数据库**：图片上传到服务器本地或 OSS，数据库只存 URL。
2. **警惕 XSS 攻击**：渲染 Markdown 时一定要做 XSS 过滤（如使用 `DOMPurify`）。
3. **不要自己写富文本编辑器**：老老实实用 Markdown 或引入开源编辑器组件。
4. **先跑通，再完美**：第一版 UI 丑一点没关系，核心逻辑跑通后再美化。

## 第三部分：Vue3 + Spring Boot 3 专属实战指南

**Vue + Java (Spring Boot)** 是国内企业级开发最经典的组合。用这套技术栈手写博客，能彻底搞懂前后端分离的精髓。

### 1. 技术栈精选清单
#### 🛠️ 后端 (Java)
- **核心框架**：Spring Boot 3.x + JDK 17
- **ORM 框架**：MyBatis-Plus
- **数据库**：MySQL 8.0
- **接口文档**：Knife4j
- **鉴权**：手写简单的 JWT 拦截器（练手项目不建议上 Spring Security）
- **工具库**：Hutool、Lombok

#### 🎨 前端 (Vue)
- **核心框架**：Vue 3 (`<script setup>`) + Vite
- **路由与状态**：Vue Router 4 + Pinia
- **UI 组件库**：Element Plus (后台) + Tailwind CSS (前台)
- **Markdown 核心库**：`md-editor-v3` (编辑器与渲染器)
- **网络请求**：Axios

### 2. 数据库设计 SQL
```sql
-- 1. 管理员用户表
CREATE TABLE `sys_user` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `username` varchar(50) NOT NULL COMMENT '用户名',
  `password` varchar(100) NOT NULL COMMENT '密码(加密后)',
  `avatar` varchar(255) DEFAULT NULL COMMENT '头像',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

-- 2. 文章表
CREATE TABLE `article` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `title` varchar(200) NOT NULL COMMENT '文章标题',
  `summary` varchar(500) DEFAULT NULL COMMENT '文章摘要',
  `content` longtext NOT NULL COMMENT 'Markdown 原文',
  `cover_image` varchar(255) DEFAULT NULL COMMENT '封面图 URL',
  `status` tinyint DEFAULT '1' COMMENT '状态: 0-草稿, 1-已发布',
  `view_count` int DEFAULT '0' COMMENT '阅读量',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文章表';
```
